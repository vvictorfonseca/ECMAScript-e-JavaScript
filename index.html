ECMAScript e JavaScript

O que é a ECMA?

Ecma International (oficialmente Associação Européia dos Fabricantes de Computadores) é uma organização sem fins lucrativos que desenvolve os padrões para hardware de computadores, comunicações e linguagens de programação.

Na web ela é famosa por ser a organização que mantém a especificação ECMAScript , que é a especificação núcleo para a linguagem JavaScript .

Qual a diferença entre ECMAScript e JavaScript?

Há pessoas que defendem que estamos atualmente utilizando ECMAScript, e que o nome JavaScript é usado apenas por costume, já que esse nome pertence à Oracle. Em contrapartida há pessoas que dizem que é tudo a mesma coisa.

Se formos definir que há uma diferença, poderíamos dizer que o ECMAScript é a especificação, e que JavaScript é uma implementação por parte dos navegadores.

ES6 - Default Parameter

Inicializar parâmetros de função desta maneira tornará a leitura delas mais fácil e menos propensa a erros.

ES6 - Template Strings

Template Strings são strings que permitem expressões embutidas. Você pode utilizar string multi-linhas e interpolação de string com elas.

Basicamente é uma nova forma de criar strings e tornar o seu código um pouco mais legível.

Template strings são envolvidas por crases (` `) em vez de aspas simples ou duplas. Template strings podem possuir placeholders. Estes são indicados por um cifrão seguido de chaves (${expression}). As expressões nos placeholders, bem como o texto em volta delas são passados a uma função. A função padrão apenas concatena as partes em uma string única.

O que é strict mode?

Basicamente a opção nos possibilita colocar um programa ou uma função para que ela opere em modo estrito (‘strict’). Esse contexto estrito previne algumas ações de serem tomadas e envia mais exceções. A linha use strict instrui o navegador para usar o modo Estrito, que é como se fosse uma versão mais segura e reduzida do JavaScript.

Declarando o modo estrito

O modo estrito é declarado adicionando "use strict"; para o início de um script ou uma função. Declarado no início de um script, tem escopo global (todo o código no script será executado em modo estrito

Declarado dentro de uma função, tem escopo local (somente o código dentro da função está em modo estrito):

O que é Hoisting?

Hoisting foi pensado como uma maneira geral de pensar sobre como os contextos de execução (especificamente as fases de criação e execução) funcionam em JavaScript.

Conceitualmente, por exemplo, uma definição estrita de elevação sugere que as declarações de variáveis e funções são fisicamente movidas para o topo do seu código, mas isso não é realmente o que acontece. Em vez disso, as declarações de variável e função são colocadas na memória durante a fase de compilação, mas permanecem exatamente onde você as digitou no seu código.

Qual a diferença de var, let e const?

var

- Antes do ES6 o var era a única forma de declarar variáveis

- Declarações com var tem escopo global ou escopo de função/local.

- Variáveis de var podem ser declaradas de novo e atualizadas

Problemas com var

- Problema quando você não perceber que uma variável greeter já havia sido definida antes.

let

- let pode ser atualizado, mas não declarado novamente.

- let tem escopo de bloco, Um bloco é uma porção de código cercado por {}. Um bloco vive dentro dessas chaves. Tudo o que estiver cercado por chaves é um bloco. Assim, uma variável declarada com let em um bloco estará disponível apenas dentro daquele bloco.

const

- Variáveis declaradas com const mantêm valores constantes.

- const não pode ser atualizado nem declarado novamente

Qual a diferença entre for, forEach e for.. of ?

for

- Ele permite repetir um trecho de código um determinado número de vezes, geralmente com base em um contador e uma condição de parada baseada no valor desse contador.

- Fornecem acesso ao índice do array, não ao elemento real.

forEach

- Com o forEach() você tem acesso ao próprio elemento do array.
